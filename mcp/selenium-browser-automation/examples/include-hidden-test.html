include_hidden Parameter Test Suite

Tests for get_aria_snapshot(selector, include_hidden=True|False). Default (include_hidden=false) excludes hidden content. When include_hidden=true, hidden elements appear with [hidden:X] markers.

Test 1: aria-hidden="true"

Elements with aria-hidden="true" are removed from accessibility tree. Marker: [hidden:aria]

This content is aria-hidden

Test 2: inert attribute

Elements with inert attribute are removed from accessibility tree and non-interactive. Marker: [hidden:inert]

Inert Link

Test 3: CSS display:none

Elements with display:none are not rendered. Marker: [hidden:css]

Test 4: CSS visibility:hidden

Elements with visibility:hidden take up space but are invisible. Marker: [hidden:css]

Test 5: CSS opacity:0

Elements with opacity:0 are transparent (technically in accessibility tree, but visually hidden). Marker: [hidden:css]

Test 6: Combined Hiding

Element hidden by both aria-hidden="true" AND display:none. Marker: [hidden:aria,css] or [hidden:css,aria]

Test 7: Screen-reader-only (NOT hidden)

.sr-only pattern (clip, offscreen) should NOT be detected as hidden. These elements ARE in the accessibility tree - they're just visually hidden. No [hidden:X] marker should appear.

Test 8: Nested Hiding

Parent has aria-hidden="true", child has display:none. Parent marker: [hidden:aria], child marker: [hidden:css]

Test 9: HTML5 hidden attribute

The hidden attribute applies display:none by default. Marker: [hidden:css]

Test 10: Compact Tree Interaction

Hidden containers should NOT be collapsed by compact_tree rules. When include_hidden=true AND compact_tree=true, the generic wrapper div should be preserved (not collapsed) because it has [hidden:aria].

Test 11: aria-labelledby references hidden element

Per W3C, aria-labelledby MUST access hidden elements for name computation. The button should have accessible name "Hidden Label Text" regardless of include_hidden setting.
