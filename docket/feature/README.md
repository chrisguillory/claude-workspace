# Feature

A net-new capability you're **interested in building** — the *what* is clear enough to name, even
if you may never get to it. Capturing it parks the intent where the code lives, so a future you (or
an agent) can pick it up without rebuilding the context.

## What to capture

A pointer + headline, not a spec — the depth stays recoverable from the indexed session transcript
(the skill appends a session-provenance footer). `area` + entry points are load-bearing: they make
`rg -l 'area: X' docket/` surface every local intent when you next touch that subsystem.

```markdown
---
area: selenium-browser
title: full-page screenshot via scroll-and-stitch
---

**What.** The capability, in a line or two.

**Why.** What it unlocks — the motivation for wanting it.

**Sketch.** A rough direction or the entry points it'd touch — orientation, not a spec.
```

## Ask before filing

- What's the capability, stated as the outcome it gives you — not the implementation?
- What does it unlock that's awkward or impossible today?
- Which subsystem (`area`) would own it, and where would it hook in?
